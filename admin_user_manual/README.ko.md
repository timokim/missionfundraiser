## 이 앱으로 할 수 있는 일

이 앱은 교회나 행사 관리자가 펀드레이저용 공개 주문서를 만들고 운영할 수 있도록 도와줍니다.

**관리자**는 다음을 할 수 있습니다.

- 매직 링크로 로그인 (비밀번호 없음)
- 여러 펀드레이저 생성 및 관리
- 제목, 설명, 대표 이미지, e-transfer 이메일, 안내 문구 설정
- 가격, 수량 제한, 이미지가 있는 메뉴 아이템 추가
- 주문자 정보를 받는 커스텀 폼 필드 추가
- **사전주문** 링크와 **현장주문** 링크를 각각 공유
- Draft / Published / On-site / Closed 네 가지 상태로 주문 접수 제어
- 공동 관리자 초대
- 주문 목록 확인 (필터, 정렬, 매출 집계)
- 주문별 입금 여부 표시
- 잘못된 주문 삭제
- CSV로 주문보내기
- 로그인 없이 볼 수 있는 읽기 전용 주문 현황 링크 공유

**주문자**(로그인 불필요)는 다음을 할 수 있습니다.

- 사전주문 폼으로 주문 제출
- 현장주문 폼으로 주문 제출
- 제출 후 주문 확인 페이지 보기

## URL과 페이지

| URL | 대상 | 용도 |
|-----|------|------|
| `/` | 누구나 | 메인 페이지 (로그인 버튼) |
| `/login` | 관리자 | 매직 링크 로그인 |
| `/admin/fundraisers` | 관리자 | 펀드레이저 목록 |
| `/admin/fundraisers/[id]` | 관리자 | 펀드레이저 편집 |
| `/admin/fundraisers/[id]/orders` | 관리자 | 주문 스프레드시트 |
| `/admin/fundraisers/[id]/orders/[orderId]` | 관리자 | 주문 상세 |
| `/f/[publicId]` | 공개 | 사전주문 폼 |
| `/f/[publicId]/onsite` | 공개 | 현장주문 폼 |
| `/f/[publicId]/confirmation/[orderId]` | 공개 | 주문 제출 후 확인 |
| `/f/[publicId]/orders` | 공개 | 읽기 전용 주문 현황 (공유 링크) |

`/admin/*` 경로는 로그인이 필요합니다. `/f/*` 공개 경로는 로그인이 필요 없습니다.

## 관리자 사용 흐름 요약

일반적인 순서:

1. 매직 링크로 로그인
2. 새 펀드레이저 생성
3. 기본 정보, 메뉴, 폼 필드 설정
4. **Published**로 변경 후 사전주문 링크 공유
5. 주문 페이지에서 주문 확인 및 **Paid** 체크
6. 행사 당일 **On-site**로 변경 후 현장주문 링크 공유
7. 필요 시 주문 현황 공유 링크 전달
8. 공동 관리자 추가
9. **Closed**로 마감

## 로그인 및 로그아웃

### 로그인

1. 앱 첫 화면에서 **로그인** 버튼을 누릅니다.
2. 이메일 주소를 입력합니다.
3. **Send magic link** 버튼을 누릅니다.
4. 이메일 받은편지함에서 링크를 클릭합니다.
5. `/admin/fundraisers` 페이지(또는 원래 가려던 페이지)로 이동합니다.

비밀번호는 없습니다. 이메일로 받은 일회용 링크로 로그인합니다.

로그인에 실패하면 로그인 페이지에 **Sign-in failed. Try again.** 메시지가 표시될 수 있습니다.

### 관리자 헤더

로그인 후 상단 헤더에 다음이 보입니다.

- 앱 제목 (큰빛 다운타운 선교 펀드레이징)
- 본인 이메일 주소
- **Sign out** 버튼

### 로그아웃

헤더의 **Sign out** 버튼을 누르면 로그아웃됩니다.

## 펀드레이저 목록 페이지

**Your fundraisers** 페이지에서 볼 수 있는 것:

- 내가 소유한 펀드레이저
- 공동 관리자로 공유받은 펀드레이저 (**(shared)** 표시)
- 각 펀드레이저의 상태 뱃지 (Draft, Published, On-site, Closed)
- 주문 수

가장 최근에 수정한 펀드레이저가 위에 표시됩니다.

이 페이지에서 할 수 있는 일:

- **New fundraiser** 로 새 펀드레이저 생성
- 펀드레이저 행을 클릭해 편집 페이지 열기

목록 로드에 실패하면 환경 변수, migration, Supabase 스키마 설정 안내가 포함된 오류 메시지가 표시됩니다.

UI에서 펀드레이저를 삭제하는 기능은 없습니다.

- 제목: `Untitled fundraiser`

생성 직후 편집 페이지로 이동합니다.

## 기본 정보 설정

편집 페이지 **Basics** 섹션에서 핵심 설정을 합니다.

대부분의 필드는 **입력 후 다른 곳을 클릭하면 자동 저장**됩니다 (blur 시 저장). 상태(Status)는 드롭다운에서 선택하면 즉시 저장됩니다.

### 제목

관리자 페이지와 공개 주문 페이지에 표시됩니다.

### 설명

**사전주문** 공개 폼 상단에만 표시됩니다. 현장주문 폼에는 표시되지 않습니다.

펀드레이저 목적, 픽업 안내, 주문자에게 필요한 공지 등을 적으면 좋습니다.

### E-transfer 이메일

가격이 있는 메뉴가 하나라도 있으면 사전주문 폼에서 총액 옆에 표시됩니다.

사전주문 폼에는 **"{이메일}으로 e-transfer 부탁드립니다!"** 한국어 문구와 함께 보입니다.

### 대표 이미지 (Hero image)

펀드레이저 대표 사진을 업로드합니다. **사전주문** 폼 상단에만 표시됩니다.

이미지는 Supabase storage (`fundraiser-assets`)에 저장됩니다.

### 상태

[펀드레이저 상태](#펀드레이저-상태) 섹션을 참고하세요.

### 마감 메시지 (Message when closed)

**사전주문** 링크에서 주문이 닫혔을 때 보이는 안내 문구입니다 (`on_site` 또는 `closed` 상태에서 사전주문 링크 접속 시).

비워두면 기본 영어 문구가 표시됩니다: *"This order form is no longer accepting responses."*

### 주문 확인 문구 (Order confirmation text)

주문 제출 후 **확인 페이지** 상단 (**⚠️ 안내 ⚠️**)에 표시되는 선택 문구입니다.

입금 안내, 픽업 방법, 다음 단계 등을 적을 수 있습니다.

## 펀드레이저 상태

펀드레이저는 **네 가지** 상태가 있습니다.

| 상태 | UI 표시 | 의미 |
|------|---------|------|
| `draft` | Draft (not public) | 아직 공개되지 않음 |
| `published` | Published (pre-order and on-site open) | 사전주문 + 현장주문 둘 다 주문 가능 |
| `on_site` | On-site (pre-order closed, on-site open) | 사전주문 닫힘, 현장주문만 열림 |
| `closed` | Closed (read-only public page) | 두 링크 모두 새 주문 불가 |

### 상태별 링크 동작

| 상태 | 사전주문 `/f/...` | 현장주문 `/f/.../onsite` | 주문 현황 `/f/.../orders` |
|------|-------------------|--------------------------|---------------------------|
| Draft | **404** (없음) | **404** | **404** |
| Published | 열림 | 열림 | 보임 |
| On-site | 닫힘 (마감 메시지) | 열림 | 보임 |
| Closed | 닫힘 (마감 메시지) | 닫힘 (고정 한국어 메시지) | 보임 |

### 언제 어떤 상태를 쓸까요?

- **Draft** — 관리자 화면에서만 준비할 때
- **Published** — 사전주문과 현장주문을 모두 받을 때
- **On-site** — 행사 당일: 사전주문은 끄고 현장주문만 받을 때
- **Closed** — 주문 접수 완전 종료 (페이지는 유지)

편집기의 마감 메시지 아래에 Published / On-site / Closed가 두 주문 링크에 어떻게 영향을 주는지 힌트가 표시됩니다.

## 공개 링크

펀드레이저 편집 페이지 상단:

| 버튼 | 복사되는 URL |
|------|-------------|
| **Copy public link** | 사전주문 폼: `/f/{publicId}` |
| **Copy 현장주문 link** | 현장주문 폼: `/f/{publicId}/onsite` |
| **View orders →** | 관리자 주문 페이지로 이동 |

클립보드 복사가 실패하면 URL이 페이지 안 메시지로 표시될 수 있습니다.

### 중요 사항

- **Draft** 상태에서 복사한 링크는 공개 접속 시 **404**입니다. 최소 **Published**, **On-site**, **Closed** 중 하나여야 공개 페이지가 열립니다.
- **Closed** 상태에서도 페이지 URL은 열리지만 새 주문은 받지 않습니다.
- 공동 관리자도 소유자와 같이 링크를 복사·공유할 수 있습니다.

## 메뉴 아이템

**Items** 섹션에서 **Add item** 으로 메뉴를 추가합니다 (기본 이름: `New item`).

### 항목별 설정

| 항목 | 설명 |
|------|------|
| **Name** | 주문자에게 보이는 이름 |
| **Unit label** | 단위 (예: `box`, `jar`) — 가격·재고 옆에 표시 |
| **Description** | 공개 폼에 보이는 설명 |
| **Quantity cap** | 최대 판매 수량; 비우면 무제한 |
| **Price (CAD)** | 선택; 비우면 주문은 되지만 총액에 포함 안 됨 |
| **Active on public form** | 체크 해제 시 새 주문에서 숨김 (기존 주문 기록은 유지) |
| **Item image** | 메뉴별 사진 |

### 재고 및 품절

- 남은 수량 = 수량 제한 − 이미 주문된 수량 (사전주문 + 현장주문 합산)
- **사전주문** 폼: 품절 항목은 회색 **Sold out**으로 계속 표시
- **현장주문** 폼: 품절 항목은 **목록에서 숨김**
- 주문 중 재고가 바뀌면 제출 실패 후 새로고침 안내가 나올 수 있음

### 메뉴 순서

각 항목의 **Up** / **Down** 버튼으로 순서를 바꿉니다. 공개 폼도 이 순서를 따릅니다.

### 메뉴 삭제

**Remove Menu Item** 으로 완전 삭제합니다.

실제 주문이 들어온 뒤에는 삭제보다 **Active on public form** 체크 해제를 권장합니다.

## 주문 폼 필드

**Order form fields** 섹션에서 **Add field** 로 필드를 추가합니다.

이 필드는 **사전주문** 폼의 **Your details** 영역에만 표시됩니다. **현장주문** 폼에는 커스텀 필드가 없고 **이름(Name)** 만 필수로 받습니다.

### 필드 설정

| 설정 | 설명 |
|------|------|
| **Label** | 주문자에게 보이는 이름 |
| **Key (CSV column)** | 내부 저장 키; 저장 시 공백은 `_`로 변환. 주문 표·CSV에 사용 |
| **Type** | `text`, `email`, `phone`, `textarea`(Paragraph), `select`(Dropdown) |
| **Required** | 필수 여부 |
| **Options** | select 타입: 한 줄에 하나씩 선택지 |

새 필드는 라벨에서 자동으로 key가 생성됩니다 (중복 시 `key_1` 등).

### 필드 순서

필드는 생성 시 `sort_order`로 정렬됩니다. 메뉴와 달리 필드에는 **Up/Down 순서 변경 버튼이 없습니다**.

### 필드 삭제

**Remove field** 로 삭제합니다. 주문이 많이 쌓인 뒤 구조 변경은 신중히 하세요.

### Service no 필터

key를 `service_no`로 맞추면 주문 페이지 **Service no** 필터가 동작합니다.

## 공동 관리자

**Collaborators** 섹션에서 **소유자만** 이메일로 다른 관리자를 초대할 수 있습니다.

### 추가 방법

1. 이메일 입력
2. **Add** 클릭

상대방이 한 번도 로그인한 적이 없으면 *"No account with that email..."* 오류가 표시됩니다.

### 권한 비교

| 작업 | 소유자 | 공동 관리자 |
|------|--------|------------|
| 기본 정보·메뉴·필드 수정 | ✓ | ✓ |
| 주문 확인·Paid·삭제 | ✓ | ✓ |
| 공개/현장 링크 복사 | ✓ | ✓ |
| 공동 관리자 추가/제거 | ✓ | ✗ |

공유받은 펀드레이저는 목록에 **(shared)** 로 표시됩니다.

### 제거

소유자만 같은 화면에서 공동 관리자를 제거할 수 있습니다.

## 공개 사전주문 폼

URL: `/f/{publicId}`

열려 있을 때 (상태 `published`, 또는 닫힌 경우 `on_site`/`closed`의 읽기 전용 화면):

**상단**

- 대표 이미지 (있을 경우)
- 제목
- 설명

**Items**

- 활성 메뉴 그리드 (사진, 설명, 가격, `N {단위} 남음` 또는 **Unlimited**)
- +/- 수량 조절
- 품절 시 회색 **Sold out**

**Your details**

- 설정한 모든 폼 필드

**결제 안내** (가격 있는 메뉴가 있을 때)

- e-transfer 이메일 + 한국어 안내
- CAD 총액

**제출**

- **Submit order** 버튼
- 중복 제출 방지 (같은 세션에서 실수로 두 번 눌러도 동일 주문 처리)

### 주문자가 볼 수 있는 오류

| 상황 | 메시지 |
|------|--------|
| 품절/재고 변경 | 새로고침 후 다시 시도 |
| 필수 항목 누락 | 필수 필드 입력 안내 |
| 메뉴 미선택 | 수량 1개 이상 선택 안내 |
| 기타 | 서버 또는 일반 오류 메시지 |

## 공개 현장주문 폼 (현장주문)

URL: `/f/{publicId}/onsite`

행사 현장에서 빠르게 주문받기 위한 폼입니다. 사전주문과의 차이:

| 기능 | 사전주문 | 현장주문 |
|------|--------|----------|
| 대표 이미지 | 표시 | 없음 |
| 설명 | 표시 | 없음 |
| 커스텀 폼 필드 | 모두 표시 | **이름만** (필수) |
| 품절 메뉴 | 회색 표시 | **숨김** |
| e-transfer 안내 문구 | 표시 | 없음 (총액은 표시) |
| 제목 영역 | 제목 | 제목 + **현장주문** |

주문자 이름은 `name` 필드에 저장됩니다.

### 현장주문이 닫혔을 때

현장주문을 받지 않는 상태(`closed` 등)에서는 관리자가 설정한 마감 메시지가 아니라 **앱에 고정된 한국어/라오어 안내 문구**가 표시됩니다.

## 주문 확인 페이지

URL: `/f/{publicId}/confirmation/{orderId}`

주문 제출 성공 후 이동하는 페이지입니다.

표시 내용:

1. **⚠️ 안내 ⚠️** — 편집기의 **Order confirmation text** (설정한 경우)
2. 펀드레이저 제목, **Order Confirmation** 제목
3. 주문자 이름 (있을 경우 — 현장주문은 항상)
4. **Total Price** (큰 글씨)
5. **Order Summary** — 메뉴별 수량, 단위, 금액

현재 주문자에게 주문 ID를 크게 보여주거나 폼으로 돌아가는 버튼은 없습니다.

## 주문 관리

편집 페이지 **View orders →** 로 접속합니다.

### 스프레드시트

한 행 = 한 주문. 열 구성:

- **Submitted** — 제출 시간 (정렬 가능)
- **Order** — 주문 ID (클릭 시 상세, 정렬 가능)
- **Paid** — 입금 체크박스 (정렬 가능)
- **Total (CAD)** (정렬 가능)
- 폼 필드 key별 열 (정렬 가능)
- **Ordered items** — 요약 텍스트 (정렬 가능)
- 메뉴별 수량 열 (정렬 가능)

열 헤더를 클릭하면 오름차순/내림차순 정렬됩니다.

### 입금 여부 (Paid)

목록 또는 상세 페이지에서 체크합니다. 저장 후 새로고침해도 유지됩니다.

공개 주문 현황 페이지에서는 읽기 전용으로만 표시됩니다.

### 필터

- **Menu item** — 해당 메뉴가 포함된 주문만 표시; 메뉴 열도 좁혀짐
- **Service no** — `service_no` key 필드 값으로 필터

### CSV 다운로드

**Download CSV** 는 **현재 필터가 적용된** 표를보냅니다.

열: `submitted_at`, `order_id`, `total_cad`, 필드 key들, `ordered_items`, 메뉴별 수량.

**Paid 열은 CSV에 포함되지 않습니다.**

공개 주문 현황 페이지에서도 동일한 CSV 다운로드가 가능합니다.

### 공유 가능한 주문 링크

**Copy share link** → `/f/{publicId}/orders` 복사.

링크를 받은 사람은 **로그인 없이** 주문 목록을 볼 수 있습니다.

할 수 있는 것:

- 스프레드시트·매출 집계 보기
- 필터·정렬
- CSV 다운로드

할 수 없는 것:

- Paid 수정
- 주문 삭제
- 펀드레이저 설정 변경
- 관리자 주문 상세 페이지 열기 (공개 페이지에서는 주문 ID가 링크가 아님)

주문자 개인정보가 포함되므로 신뢰할 수 있는 사람에게만 공유하세요.

### 매출 집계

표 아래에 메뉴별 판매 수량, 매출, **Total** 합계가 표시됩니다.

## 주문 상세 페이지

관리자 주문 목록에서 주문 ID를 클릭합니다.

표시 내용:

- 주문 ID, 제출 시간, 총액(CAD), 입금 여부
- 모든 폼 응답
- 주문 메뉴와 수량

가능한 작업:

- **Paid** 체크/해제
- **Remove order** — 확인 후 삭제, **되돌릴 수 없음**

삭제하면 목록과 매출 집계에서 제외됩니다.

## 공개 주문 현황 페이지 (공유 링크)

URL: `/f/{publicId}/orders`

관리자 주문 페이지와 같은 레이아웃이지만:

- Paid는 읽기 전용
- 주문 ID는 링크가 아님
- **Copy share link** 버튼 없음
- **← Back to fundraiser** 로 사전주문 폼으로 돌아감

상태가 `published`, `on_site`, `closed` 일 때 접근 가능 (`draft`는 404).

## 추천 운영 순서

1. 펀드레이저 생성
2. 제목, 설명, e-transfer 이메일, 대표 이미지 입력
3. 메뉴(가격, 수량 제한, 사진) 및 폼 필드 추가 (이름, 전화번호, `service_no` 등)
4. **Order confirmation text** 설정 (제출 후 안내가 필요하면)
5. **Published** 로 변경
6. 사전주문 링크 직접 테스트
7. **Copy public link** 공유
8. **View orders** 로 주문 확인, 입금 시 **Paid** 체크
9. 행사 당일 **On-site** + **Copy 현장주문 link** 공유
10. 필요 시 주문 페이지 **Copy share link** 로 현황만 공유
11. **Closed** 로 마감, **Message when closed** 작성

## 팁과 권장 사항

- 준비 중에는 **Draft** 유지. 공개 링크는 **Draft에서 404**입니다.
- 상태 변경 후 사전주문·현장주문 링크를 둘 다 테스트하세요.
- service number 필터를 쓰려면 key를 `service_no`로 맞추세요.
- 주문이 쌓인 뒤에는 메뉴 삭제보다 비활성화를 우선하세요.
- 공동 관리자는 한 번 로그인한 적이 있는지 확인 후 추가하세요.
- 주문 현황 공유 링크에는 개인정보가 있으니 공유 대상을 제한하세요.
- 사전주문만 끄고 현장주문을 열어두려면 **On-site** 상태를 사용하세요.
- 입금·픽업 안내는 **Order confirmation text**에 넣으면 주문 직후 바로 보입니다.

## 문제 해결

### 공동 관리자 추가가 안 될 때

- 아직 이 앱에 로그인한 적이 없음
- 이메일이 계정과 다름
- 이미 해당 펀드레이저 소유자임

### 공개 링크가 404일 때

- 아직 **Draft** 상태 — 먼저 Published 등으로 변경
- 잘못된 URL

### 사전주문 폼에서 주문이 안 될 때

| 상태 | 사전주문 |
|------|----------|
| Draft | 404 |
| Published | 주문 가능 |
| On-site | 닫힘 (마감 메시지) |
| Closed | 닫힘 (마감 메시지) |

### 현장주문 폼에서 주문이 안 될 때

| 상태 | 현장주문 |
|------|----------|
| Draft | 404 |
| Published | 주문 가능 |
| On-site | 주문 가능 |
| Closed | 닫힘 (고정 한국어 메시지) |

### 품절/재고 오류

다른 주문으로 재고가 소진됨. 페이지를 새로고침 후 다시 시도.

### Service no 필터가 비어 있을 때

key가 정확히 `service_no`인 필드가 있는지 확인.

### Paid 체크가 유지되지 않을 때

배포 환경에서 최신 DB migration 적용 여부 확인.

### 펀드레이저 목록이 안 불러와질 때

`.env.local`, migration, Supabase에서 `fundraiser_app` 스키마 노출 설정 확인 (페이지 오류 메시지 참고).
