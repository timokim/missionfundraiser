# 관리자 사용자 매뉴얼

이 문서는 펀드레이저 관리자 관점에서 앱을 사용하는 방법을 설명합니다.

다음 내용을 포함합니다.

- 로그인
- 펀드레이저 생성
- 공개 주문서 설정
- 메뉴 아이템 추가
- 주문 폼 필드 추가
- 공개 링크 공유
- 공동 관리자 추가
- 주문 확인 및 관리

## 이 앱으로 할 수 있는 일

이 앱은 교회나 행사 관리자들이 펀드레이저용 공개 주문서를 만들고 관리할 수 있도록 도와줍니다.

관리자는 다음 작업을 할 수 있습니다.

- 여러 개의 펀드레이저 생성 및 관리
- 제목, 설명, 대표 이미지, 입금 이메일 등 공개 페이지 정보 설정
- 가격, 재고 제한, 이미지를 포함한 메뉴 아이템 추가
- 주문자 정보를 받기 위한 커스텀 폼 필드 추가
- 공개 주문 링크 발행 및 공유
- 펀드레이저를 마감 상태로 전환하여 주문은 막고 페이지는 유지
- 공동 관리자 추가
- 주문 목록 및 주문 상세 확인
- 주문별 입금 여부 표시
- 필요 시 주문 삭제

## 관리자 사용 흐름 요약

일반적인 사용 순서는 다음과 같습니다.

1. 로그인합니다.
2. 새 펀드레이저를 만듭니다.
3. 기본 정보를 입력합니다.
4. 메뉴 아이템을 추가합니다.
5. 주문 폼 필드를 추가합니다.
6. 상태를 `Published`로 변경합니다.
7. 공개 주문 링크를 복사해서 공유합니다.
8. 주문 페이지에서 주문을 확인합니다.
9. 입금이 들어오면 `Paid`를 체크합니다.
10. 필요하면 공동 관리자를 추가합니다.
11. 주문을 더 이상 받지 않을 때 상태를 `Closed`로 변경합니다.

## 로그인

앱 첫 화면에서 `로그인` 버튼을 누릅니다.

로그인 후에는:

- `/admin/fundraisers` 페이지로 이동합니다
- 본인이 만든 펀드레이저 목록이 보입니다
- 다른 사람이 공동 관리자로 추가한 펀드레이저도 함께 보이며 `(shared)` 표시가 붙습니다

## 펀드레이저 목록 페이지

`Your fundraisers` 페이지에서는 다음을 볼 수 있습니다.

- 내가 소유한 펀드레이저
- 내가 공동 관리자로 참여 중인 펀드레이저
- 각 펀드레이저의 현재 상태
- 현재 주문 수

이 페이지에서 할 수 있는 일:

- `New fundraiser` 버튼으로 새 펀드레이저 생성
- 기존 펀드레이저를 눌러 편집 페이지 열기

## 새 펀드레이저 만들기

`New fundraiser` 버튼을 누르면 새 펀드레이저가 즉시 생성됩니다.

기본값은 다음과 같습니다.

- 제목: `Untitled fundraiser`
- 상태: `Draft`

생성 직후 해당 펀드레이저의 편집 페이지로 이동합니다.

## 기본 정보 설정

펀드레이저 편집 페이지 상단에서 기본 정보를 설정할 수 있습니다.

### 제목

제목은 다음 위치에 표시됩니다.

- 관리자 페이지
- 공개 주문 페이지

### 설명

공개 주문 페이지 상단에 표시됩니다.

다음과 같은 내용을 적으면 좋습니다.

- 펀드레이저 목적
- 수령 또는 전달 안내
- 주문자에게 필요한 공지사항

### E-transfer 이메일

가격이 있는 메뉴가 하나라도 있으면 공개 주문 페이지에서 총액과 함께 이 이메일이 표시됩니다.

주문자는 이 이메일로 e-transfer를 보낼 수 있습니다.

### 대표 이미지

펀드레이저 대표 이미지를 업로드할 수 있습니다.

이 이미지는 공개 페이지 상단에 표시됩니다.

### 상태

펀드레이저 상태는 세 가지입니다.

- `Draft`: 공개되지 않음
- `Published`: 공개되며 주문 가능
- `Closed`: 공개 페이지는 보이지만 새 주문은 불가

### 마감 메시지

상태가 `Closed`일 때 방문자는 제목, 이미지, 설명은 계속 볼 수 있지만 주문은 할 수 없습니다.

이때 표시할 안내 문구를 입력할 수 있습니다.

예:

- 주문이 마감되었음을 알리는 문구
- 픽업 안내
- 감사 메시지

## 공개 링크와 주문 링크

편집 페이지 상단에는 다음 버튼이 있습니다.

- `View orders`
- `Copy public link`

### Copy Public Link

주문자용 공개 주문 링크를 복사합니다.

이 링크를 통해 주문자는:

- 메뉴를 확인하고
- 수량을 선택하고
- 폼을 작성한 뒤
- 주문을 제출할 수 있습니다

중요:

- 공개 링크는 펀드레이저가 `Published` 또는 `Closed` 상태일 때 의미가 있습니다
- `Draft` 상태는 공개 전 준비 단계입니다

### View Orders

해당 펀드레이저의 관리자 주문 페이지로 이동합니다.

## 메뉴 아이템 추가 및 관리

`Items` 섹션에서 `Add item` 버튼을 누르면 새 메뉴 아이템을 추가할 수 있습니다.

각 메뉴 아이템은 다음 항목을 가집니다.

### Name

주문자에게 보이는 메뉴 이름입니다.

예:

- 김치
- 만두
- 쿠키 박스

### Unit Label

단위를 설명하는 선택 입력값입니다.

예:

- `box`
- `jar`
- `plate`

가격이나 재고 안내 옆에 함께 표시될 수 있습니다.

### Description

메뉴 설명입니다.

예:

- 맛이나 구성 설명
- 재료 안내
- 픽업 관련 안내

### Quantity Cap

비워두면 수량 제한이 없습니다.

숫자를 입력하면 해당 메뉴의 총 판매 가능 수량이 제한됩니다.

다음 상황에 유용합니다.

- 수량이 한정되어 있을 때
- 선주문 가능 수량을 제한하고 싶을 때

### Price (CAD)

캐나다 달러 기준 가격입니다. 선택 항목입니다.

가격이 비어 있으면:

- 주문은 가능하지만
- 총액 계산에는 포함되지 않습니다

가격이 있으면:

- 공개 주문 페이지에서 총액이 자동 계산됩니다
- 입금 이메일과 총액이 함께 표시됩니다

### Active on Public Form

체크되어 있으면:

- 공개 주문 페이지에 표시됩니다

체크 해제하면:

- 새 주문에서는 숨겨집니다
- 기존 주문 기록에는 그대로 남습니다

이미 판매를 중단했지만 기존 주문 기록은 유지하고 싶을 때 유용합니다.

### Item Image

메뉴별 이미지를 업로드할 수 있습니다.

공개 페이지에서 메뉴 옆에 표시됩니다.

### 메뉴 순서 변경

`Up`, `Down` 버튼으로 메뉴 순서를 조정할 수 있습니다.

공개 주문 페이지도 이 순서를 따릅니다.

### 메뉴 삭제

`Remove Menu Item` 버튼으로 메뉴를 완전히 삭제할 수 있습니다.

주의:

- 삭제는 비활성화보다 더 강한 동작입니다
- 이미 실제 주문이 들어온 뒤에는 완전 삭제보다 `Active on public form` 체크 해제를 먼저 고려하는 것이 더 안전합니다

## 주문 폼 필드 추가 및 관리

`Order form fields` 섹션에서 `Add field`를 눌러 주문자가 입력해야 할 필드를 추가할 수 있습니다.

이 필드들은 공개 주문 페이지의 `Your details` 영역에 표시됩니다.

각 필드는 다음 정보를 가집니다.

- 라벨
- 내부 키
- 필드 타입
- 선택형 필드 옵션
- 필수 여부
- 순서

### 자주 사용하는 필드 예시

- 이름
- 전화번호
- 이메일
- 예배부서 또는 service number
- 요청사항

### 지원하는 필드 타입

- `text`
- `email`
- `phone`
- `textarea`
- `select`

### 필수 여부

필수로 설정하면 주문자는 해당 항목을 반드시 입력해야 제출할 수 있습니다.

### Select 필드

타입이 `select`인 경우 선택 가능한 옵션 목록을 설정할 수 있습니다.

다음과 같은 경우에 유용합니다.

- service number
- 픽업 시간
- 부서나 팀 이름

### Field Key

각 필드는 내부적으로 저장되는 key 값을 가집니다.

주문을 받기 시작한 뒤에는 가능한 한 key를 자주 바꾸지 않는 것이 좋습니다.

특히 주문 페이지에서 필터로 활용할 항목은 안정적으로 유지해야 합니다.

예:

- `service_no`라는 key를 사용하면 주문 페이지의 서비스 번호 필터와 연결하기 좋습니다

### 필드 순서 변경 및 삭제

필드 순서를 조정하거나 삭제할 수 있습니다.

다만 주문이 많이 쌓인 뒤에는 구조를 크게 바꾸기 전에 영향을 먼저 확인하는 것이 좋습니다.

## 공동 관리자 추가

`Collaborators` 섹션에서 다른 관리자를 초대할 수 있습니다.

### 누가 공동 관리자를 추가할 수 있나요?

펀드레이저 소유자만 공동 관리자를 추가하거나 제거할 수 있습니다.

### 추가 방법

이메일 주소를 입력하고 `Add`를 누릅니다.

중요:

- 초대하려는 사람은 이 앱에 이미 계정이 있어야 합니다
- 한 번도 로그인한 적이 없는 사람은 먼저 가입 또는 로그인 한 번을 해야 합니다

### 공동 관리자가 할 수 있는 일

공동 관리자는 다음 작업을 할 수 있습니다.

- 관리자 목록에서 해당 펀드레이저 열기
- 기본 정보 수정
- 메뉴 아이템 수정
- 주문 폼 필드 수정
- 주문 확인 및 관리

### 공동 관리자 제거

소유자는 같은 화면에서 공동 관리자를 제거할 수 있습니다.

## 공개 주문 페이지에서 주문자가 보는 내용

펀드레이저가 `Published` 상태이면 공개 링크에서 다음 내용을 볼 수 있습니다.

- 펀드레이저 제목
- 대표 이미지
- 설명
- 주문 가능한 메뉴 아이템
- 사용자 입력 필드
- 자동 계산된 총액
- 가격이 있는 경우 e-transfer 이메일

주문자는:

- 각 메뉴 수량을 조절하고
- 정보를 입력한 뒤
- 주문을 제출할 수 있습니다

제출이 완료되면:

- 감사 메시지
- 주문 reference ID
- 최종 금액

이 표시됩니다.

## 펀드레이저 마감하기

상태를 `Closed`로 바꾸면:

- 공개 페이지는 계속 열립니다
- 새 주문은 받을 수 없습니다
- 주문 폼 대신 마감 메시지가 표시됩니다

다음과 같은 상황에 적합합니다.

- 주문 접수가 끝났을 때
- 기존 안내 페이지는 유지하고 싶을 때
- 기록은 계속 보여주고 싶을 때

## 주문 관리

편집 페이지에서 `View orders`를 눌러 주문 페이지로 이동할 수 있습니다.

주문 페이지는 스프레드시트 형태로 주문을 보여줍니다.

### 메인 스프레드시트

각 행은 하나의 주문입니다.

표에는 다음 정보가 포함됩니다.

- 제출 시간
- 주문 ID
- 입금 여부 체크박스
- 총 금액
- 폼 응답값
- 메뉴별 수량

### 입금 여부 표시

`Paid` 컬럼에서 입금 여부를 관리할 수 있습니다.

체크하면:

- DB에 저장되고
- 새로고침 후에도 유지되며
- 주문 상세 페이지에도 동일하게 반영됩니다

### 필터

주문 페이지에는 다음 필터가 있습니다.

- `Menu item`
- `Service no`

메뉴 필터를 선택하면:

- 해당 메뉴가 포함된 주문만 보이고
- 메인 표에서도 해당 메뉴 컬럼만 보입니다

서비스 번호 필터는 주문 폼에 `service_no` key를 가진 필드가 있을 때 가장 잘 동작합니다.

### CSV 다운로드

`Download CSV` 버튼으로 현재 주문 표를 CSV로 내보낼 수 있습니다.

### 공유 가능한 주문 링크

주문 페이지에는 `Copy share link` 버튼도 있습니다.

이 링크는 로그인 없이 열 수 있는 읽기 전용 주문 페이지입니다.

관리자 권한까지는 주고 싶지 않지만 주문 현황을 공유해야 할 때 유용합니다.

### 집계 테이블

스프레드시트 아래에는 메뉴별 집계 테이블이 있습니다.

여기서 확인할 수 있는 내용:

- 메뉴별 판매 수량
- 메뉴별 매출
- 전체 매출 합계

## 주문 상세 페이지

주문 ID를 클릭하면 상세 페이지로 이동합니다.

여기서는 다음 내용을 볼 수 있습니다.

- 주문 ID
- 제출 시간
- 총 금액
- 입금 여부
- 전체 폼 응답
- 주문한 메뉴와 수량

또한 다음 작업도 가능합니다.

- 입금 여부 변경
- 주문 삭제

## 주문 삭제

주문 상세 페이지에서 특정 주문을 삭제할 수 있습니다.

다음과 같은 경우에 사용할 수 있습니다.

- 중복 주문
- 테스트 주문
- 실수로 들어온 주문

삭제 후에는:

- 주문 목록에서 사라지고
- 집계 및 합계에서도 제외됩니다

## 추천 운영 순서

일반적으로는 다음 순서를 추천합니다.

1. 새 펀드레이저를 만듭니다.
2. 제목, 설명, 입금 이메일을 입력합니다.
3. 대표 이미지를 업로드합니다.
4. 메뉴 아이템과 가격, 수량 제한을 설정합니다.
5. 이름, 전화번호, `service_no` 같은 폼 필드를 추가합니다.
6. 공개 링크를 직접 열어 테스트합니다.
7. 상태를 `Published`로 바꿉니다.
8. 공개 링크를 주문자에게 공유합니다.
9. 주문 페이지에서 들어오는 주문을 확인합니다.
10. 입금이 확인되면 `Paid`를 체크합니다.
11. 필요 시 공동 관리자를 추가합니다.
12. 주문 종료 시 상태를 `Closed`로 변경합니다.

## 팁과 권장 사항

- 준비 중일 때는 `Draft` 상태를 유지하세요.
- 공개 전에 반드시 직접 공개 링크를 테스트해 보세요.
- 서비스 번호 필터를 쓰고 싶다면 필드 key를 `service_no`로 맞추는 것이 좋습니다.
- 실제 주문이 들어온 후에는 메뉴 삭제보다 비활성화를 우선 고려하세요.
- 공동 관리자를 추가하기 전에 해당 사용자가 이 앱에 한 번 로그인했는지 확인하세요.
- 관리자 권한 없이 주문 현황만 공유하려면 공개 주문 링크 또는 공유 가능한 주문 페이지를 활용하세요.

## 문제 해결

### 공동 관리자 추가가 되지 않을 때

가능한 원인:

- 상대방이 아직 앱 계정을 만들지 않았음
- 입력한 이메일이 실제 계정 이메일과 다름
- 이미 해당 펀드레이저의 소유자임

### 공개 페이지에서 주문이 되지 않을 때

상태를 확인하세요.

- `Draft`: 공개 전 상태
- `Closed`: 읽기 전용
- `Published`: 주문 가능

### Service no 필터가 비어 있을 때

주문 폼에 `service_no` key를 가진 필드가 있는지 확인하세요.

### Paid 체크가 유지되지 않을 때

배포 환경이라면 최신 DB migration이 모두 적용되었는지 확인하세요.
